THREADS
the main thread does not have to wait!
Indeed it is not blocked by pthread_join().


threads get a unique identification:
pthread_t represents the ID of a thread in UNIX-like systems
Inside a thread, you can obtain its ID using the pthread_self function.

thread functions uses the id for thread management

pthread_join indeed ensures the main program waits 
for all threads to finish before proceeding.

MUTEX - mutual exclusion
acts like a lock
It ensures that only one thread (waiter) 
can access the critical section (our register, 
the code that leads to the race condition) at a time.

initializing a mutex: 
1. Declare the mutex.
2. Call the initialization function.
3. Finally, when done, you’d invoke the destruction function

int main() 
{
    pthread_mutex_t my_mutex;
    pthread_mutex_init(&my_mutex, NULL);
    // ... other code using the mutex
    pthread_mutex_destroy(&my_mutex);
    return 0;
}

philo project:

1. input check
2. create philo struct and data struct that all philos can uses 
   & init all mutexes/forks and the philosophers and the data struct
   	- you have to init the forks before assigning them to the philosopher
	- the data struct have a pointer to all the philos and the philos have a pointer to the data struct

3. create the philo routine function
4. create the threads for the philos (OBS: when creating the threads the threads start running the function)
	- Threads run functions (e.g. make_latte( ) | serve_customer( )). 
	  So, before creating a thread, define the function it will execute.

	- to create a thread:
		int main()
		{
			pthread_t	my_thread;  
			int			status;

			status = pthread_create(&my_thread, NULL, my_function, NULL);
		}

5. monitor loop, that checks all philo time to eat compared to last 
to see if it dies or not
Things to think about:
- when to start the time
- optimizing the routine func so it doesnt take too long and gives delays
	- minimize the critical section of code (where you have mutexes active)
- when to check if a philo died
6. destroy all mutexes and free all data allocated
